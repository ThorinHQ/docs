title: "Personal MCPs"
description: "Add and manage your own MCP servers from the personal agent panel."

Personal MCPs let you connect your own [Model Context Protocol](https://modelcontextprotocol.io) servers to your personal agent without involving an admin. Each personal MCP is scoped to you — only your personal agent can see or call its tools.

Use a personal MCP when you want to give your agent access to a service that:

- Is specific to you (a personal Notion, your own scratch server, a tool only you have access to).
- Hasn't been rolled out to your whole organization yet.
- You want to test before asking an admin to install it for everyone.

For shared connectors that everyone in the org should use, ask an admin to install an [Organization MCP](#organization-mcps) instead.

## Where to find it

MCP management has moved off the user **Settings** page. The old **Integrations** tab and the MCP rows under **Resources** are gone — open your **personal agent** instead. Both **Personal MCPs** and **Organization MCPs** now live as sections on that panel.

<Note>
  If you were linking to **Settings → Integrations** or to MCP entries under an agent's **Resources** list, update those links to point at the personal agent panel.
</Note>

## Add a personal MCP

<Steps>
  <Step title="Open your personal agent">
    Go to your personal agent and scroll to the **Personal MCPs** section.
  </Step>
  <Step title="Click Add personal MCP">
    A dialog opens asking for a name and the server URL.
  </Step>
  <Step title="Enter a name and URL">
    - **Name** — a label for the connection (for example, `My Notion`).
    - **Server URL** — the HTTPS URL of your MCP server (for example, `https://mcp.example.com`).

    Only `http://` and `https://` URLs are accepted, and the host must resolve to a public address. Loopback, link-local, and private-network hosts are rejected.
  </Step>
  <Step title="Connect">
    Click **Add**. Thorin tries OAuth first and falls back to a static access token if you provided one. See [How connection works](#how-connection-works) below.
  </Step>
</Steps>

## How connection works

When you add a personal MCP, Thorin automatically picks the right auth flow for the server.

### OAuth (preferred)

If the server advertises OAuth, Thorin runs [Dynamic Client Registration (RFC 7591)](https://datatracker.ietf.org/doc/html/rfc7591) and an [authorization code + PKCE](https://datatracker.ietf.org/doc/html/rfc7636) flow on your behalf:

1. Thorin discovers the server's protected-resource and authorization-server metadata.
2. It registers Thorin as a new OAuth client with the server (no client ID or secret to copy by hand).
3. After **Add**, you're redirected to the server to sign in and approve access.
4. The server redirects back to Thorin with an authorization code, which is exchanged for tokens that are stored against your user.

If a token later expires or fails, the row shows **Expired** or **Error** and a **Reconnect** button takes you through the authorize flow again.

### Static access token (fallback)

If the server doesn't support OAuth dynamic client registration, paste a long-lived token instead:

1. In the **Add Personal MCP** dialog, expand **Static access token (fallback)**.
2. Paste the token your MCP server issued you.
3. Click **Add**.

Thorin uses the token directly as a bearer credential when calling the server. If you leave the field empty and the server has no OAuth support, the request fails with a message asking for a static token.

<Tip>
  Try OAuth first — leave the access token field blank. Thorin only falls back to the static token if discovery finds no usable OAuth configuration.
</Tip>

## Manage personal MCPs

Each row in **Personal MCPs** shows the server's name, URL, and current connection status:

| Status | Meaning |
| --- | --- |
| **Connected** | The agent can call this server right now. |
| **Expired** | OAuth token expired — click **Reconnect**. |
| **Error** | Last call failed — click **Reconnect** or **Remove**. |
| **Not connected** | The server is added but not authorized yet — click **Sign in**. |

Use the **Remove** button on any row to delete the personal MCP. Removing the server also revokes the connection. You can only remove personal MCPs that you own.

## Organization MCPs

The same panel also lists **Organization MCPs** — connectors an admin installed for everyone in the org. You can:

- **Connect** or **Reconnect** to an org MCP using its OAuth flow.
- **Test** an active connection to confirm it's healthy and see how many tools it exposes.
- **Disconnect** to revoke your personal authorization without removing the server for others.

If an org MCP shows **Setup required**, the admin still needs to finish configuring it. Ask them to complete setup before trying to connect.
