---
title: "Personal and user MCP servers"
description: "Connect MCP servers that require user scoped authorization."
---

MCP connectors let Thorin agents use external tools exposed through the [Model Context Protocol](https://modelcontextprotocol.io). Admins can add organization MCP servers, and users can connect their own account to approved servers.

Use user scoped MCP connections when an agent should act through your account or read data only you are allowed to access.

## Where to connect

Current Thorin workspaces expose user MCP connections from Settings, then Integrations. If no MCP servers appear, ask an admin to add connectors in Admin, then Integrations first.

<Note>
  Some newer builds may surface MCP controls closer to the personal agent. The current dashboard still routes user MCP authorization through Settings, then Integrations.
</Note>

## Connect an approved MCP server

<Steps>
  <Step title="Open Settings">
    Go to Settings, then Integrations.
  </Step>
  <Step title="Find the MCP server">
    Each row shows the server name, URL, and connection status.
  </Step>
  <Step title="Connect or reconnect">
    Click Connect. If the row is expired or errored, click Reconnect.
  </Step>
  <Step title="Authorize the service">
    Complete the OAuth flow for the external service.
  </Step>
  <Step title="Test the connection">
    Use Test Connection when available to confirm the connector is healthy and exposing tools.
  </Step>
</Steps>

## Connection states

| Status | Meaning |
| --- | --- |
| Connected | Thorin can call the MCP server for your account. |
| Expired | Your authorization expired. Reconnect. |
| Error | The last call failed. Reconnect or ask an admin to inspect setup. |
| Not connected | The server is available, but you have not authorized it. |
| Set up required | An admin still needs to finish connector configuration. |

## Admin installed organization MCP servers

Admins add organization MCP servers from Admin, then Integrations. They can choose a catalog entry such as Salesforce, Linear, GitHub, Notion, Sentry, HubSpot, or Figma, or add a custom MCP server URL.

Some MCP servers require admin OAuth configuration before users can connect. Salesforce Hosted MCP, for example, requires a Salesforce External Client App, PKCE, the right scopes, and Hosted MCP activation.

## How agents use MCP tools

Agents do not preload every MCP tool. They first list available MCP servers, activate the relevant server, then use the tools exposed by that server during the run. This keeps the active tool surface smaller and makes it easier to reason about which external system was used.

## Troubleshooting

- If no MCP servers appear, ask an admin to add one.
- If Connect is disabled, the connector likely needs admin setup.
- If OAuth fails, ask the admin to verify callback URLs, client settings, and scopes.
- If a connected server exposes no useful tools, test the connection and inspect the server's MCP configuration.
- If the agent cannot use the server, confirm the agent has access to the MCP resource and that the connection is active.
