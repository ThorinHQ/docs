Personal MCPs let you connect your own [Model Context Protocol](https://modelcontextprotocol.io) servers to your personal agent. Each personal MCP is scoped to you — only your personal agent can see or call its tools, and OAuth tokens are stored against your user.

Use a personal MCP when you want to give your agent access to a service that:

- Is specific to you (your Notion, your Linear, a tool only you have access to).
- Isn't covered by Thorin's [first-party connectors](/integrations/connectors).
- You want to try before recommending it to teammates.

<Note>
  MCPs in Thorin are personal-only. There is no longer an org-wide MCP catalog — admins do not install MCP servers for the tenant. Each user adds the servers they want themselves.
</Note>

## Where to find it

Personal MCPs are managed from the **chat composer**. Open any chat with your personal agent and click the **+** icon to the left of the message input. The menu has two submenus:

- **Skills** — pick a skill to run or open **Manage skills**.
- **Connectors** — opens straight into your connector list, with **Manage connectors** at the top and each of your installed servers underneath. Click **Manage connectors** (or a row that needs attention) to open the connectors dialog.

The connectors dialog is where you add new servers, sign in, reconnect, and remove connectors.

<Note>
  MCP management is no longer on the **Settings** page or the personal agent's **Resources** list. If you were linking there, update the link to the chat composer.
</Note>

## Add a connector

The connectors dialog ships with one-click presets for common servers. The current presets are **Granola**, **Notion**, **Linear**, **Sentry**, and **Figma**. Each preset has a prefilled name, URL, and logo.

<Steps>
  <Step title="Open the connectors dialog">
    In a chat with your personal agent, click the **+** icon next to the message input, hover **Connectors**, and choose **Manage connectors**.
  </Step>
  <Step title="Pick a preset">
    Click any tile under **Add a connector**. Thorin creates the personal MCP and, if the server supports OAuth, redirects you to sign in.
  </Step>
  <Step title="Approve access">
    Sign in on the provider's site and approve the scopes Thorin requests. You're returned to the chat with a success toast. The server now appears under **Your connectors** with a **Connected** badge.
  </Step>
</Steps>

## Add a custom server

For anything outside the presets — including services like GitHub, HubSpot, or Salesforce that no longer ship as built-in presets — add the MCP server by URL.

<Steps>
  <Step title="Open the connectors dialog">
    Click the **+** icon in the chat composer, hover **Connectors**, and choose **Manage connectors**.
  </Step>
  <Step title="Choose Custom">
    Click the **Custom** tile under **Add a connector**. A second dialog opens asking for a name and the server URL.
  </Step>
  <Step title="Enter a name and URL">
    - **Name** — a label for the connection (for example, `My Notion`).
    - **Server URL** — the HTTPS URL of your MCP server (for example, `https://mcp.example.com`).

    Only `http://` and `https://` URLs are accepted, and the host must resolve to a public address. Loopback, link-local, and private-network hosts are rejected.
  </Step>
  <Step title="Click Add">
    Thorin tries OAuth first and falls back to a static access token if you provided one. See [How connection works](#how-connection-works) below.
  </Step>
</Steps>

## How connection works

When you add a personal MCP, Thorin picks the right auth flow for the server automatically.

### OAuth (preferred)

If the server advertises OAuth, Thorin runs [Dynamic Client Registration (RFC 7591)](https://datatracker.ietf.org/doc/html/rfc7591) and an [authorization code + PKCE](https://datatracker.ietf.org/doc/html/rfc7636) flow on your behalf:

1. Thorin discovers the server's protected-resource and authorization-server metadata.
2. It registers Thorin as a new OAuth client with the server (no client ID or secret to copy by hand).
3. You're redirected to the server to sign in and approve access.
4. The server redirects back to Thorin, which exchanges the authorization code for tokens stored against your user.
5. You land back on the chat surface you started from, with a `?connectors=success` toast confirming the connection.

If sign-in fails, you return to the same chat with a `?connectors=error` toast and the row stays in **Not connected** state — click **Sign in** to retry.

If a token later expires or fails, the row shows **Expired** or **Error** and a **Reconnect** button takes you back through the authorize flow.

### Static access token (fallback)

If the server doesn't support OAuth dynamic client registration, paste a long-lived token instead:

1. In the **Custom** add dialog, expand **Static access token (fallback)**.
2. Paste the token your MCP server issued you.
3. Click **Add**.

Thorin uses the token directly as a bearer credential when calling the server. If you leave the field empty and the server has no OAuth support, the request fails with a message asking for a static token.

<Tip>
  Try OAuth first — leave the access token field blank. Thorin only falls back to the static token if discovery finds no usable OAuth configuration.
</Tip>

## Manage your connectors

Your installed connectors show up in two places:

- **Inline in the + menu** — under **Connectors**, each row shows the server's logo, name, and a status icon (a checkmark for **Connected**, a warning for **Expired** or **Error**). Click an unhealthy row to jump back into the OAuth flow.
- **In the connectors dialog** — the **Your connectors** section lists every personal MCP you've added with a status badge:

  | Status | Meaning |
  | --- | --- |
  | **Connected** | The agent can call this server right now. |
  | **Expired** | OAuth token expired — click **Reconnect**. |
  | **Error** | Last call failed — click **Reconnect** or remove the server. |
  | **Not connected** | The server is added but not authorized yet — click **Sign in**. |

Use the trash icon on any row to remove the connector. Removing the server also revokes the connection. You can only manage personal MCPs that you own.
