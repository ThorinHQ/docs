---
title: "Organization MCPs"
description: "Install MCP servers for everyone in your org."
---

Organization MCPs are [Model Context Protocol](https://modelcontextprotocol.io) servers an admin installs once and exposes to every user in the tenant. The server itself lives at the org level — its name, URL, and configuration are managed centrally — but each user still authorizes their own connection, so OAuth tokens and per-user data stay scoped to the person who signed in.

Use an org MCP when:

- The whole team should have access to the same tool (your shared Notion, your team's Linear workspace, your production GitHub org).
- You want consistent naming and configuration across users.
- You want to retire a connector for everyone at once.

For tools that are specific to a single user, use a [personal MCP](/integrations/personal-mcps) instead.

## Where to find it

Org MCPs are managed from **Admin → Integrations**. The page shows every connector installed for the tenant and lets admins add, configure, or remove them.

## Add from the catalog

Thorin ships a built-in catalog with prefilled URLs for the MCP servers most teams want first.

<Steps>
  <Step title="Open Admin → Integrations">
    Scroll to the **MCP catalog** section.
  </Step>
  <Step title="Pick a server">
    Catalog entries today: **Salesforce**, **Linear**, **GitHub**, **Notion**, **Sentry**, **HubSpot**, and **Figma**. The URL for each is already configured.
  </Step>
  <Step title="Click Add">
    Thorin installs the server with the prefilled URL and adds it to your org's MCP list. Each user can now connect from their personal agent panel.
  </Step>
</Steps>

## Add a custom MCP

For anything outside the catalog, install by URL.

<Steps>
  <Step title="Click Add MCP server">
    The button is at the top of the **MCP servers** section in **Admin → Integrations**.
  </Step>
  <Step title="Enter a name and URL">
    - **Name** — how the connector appears to users (for example, `Internal wiki`).
    - **Server URL** — the HTTPS URL of the MCP server.

    Only `http://` and `https://` URLs are accepted, and the host must resolve to a public address.
  </Step>
  <Step title="Save">
    The server appears in the org MCP list. Users can now connect to it from their personal agent panel.
  </Step>
</Steps>

## Configure an installed server

Each row in **Admin → Integrations** lets you:

- **Rename** the server — the new name appears in every user's panel.
- **Change the URL** — useful when a vendor moves their MCP endpoint.
- **Remove** the server — see the note below.

If a server needs additional setup before users can connect (for example, a tenant-level token the admin has to provide), the row shows a **Setup required** badge. Finish setup before asking the team to connect.

## How users connect

Installing an org MCP makes it available, but doesn't authorize anyone — that's per-user.

Each teammate opens their personal agent, finds the server under **Organization MCPs**, and clicks **Connect**. Thorin runs the OAuth flow against their account and stores the resulting tokens against their user. See [Personal MCPs → Organization MCPs](/integrations/personal-mcps#organization-mcps) for the user-side flow.

<Note>
  Removing an org MCP disconnects it for every user in the tenant — their personal authorizations are revoked along with the server. Reinstalling later requires every user to reconnect.
</Note>
