---
title: "Integrations overview"
description: "Connect Thorin to the systems agents need for context and action."
---

Integrations give Thorin agents access to external context and tools. Admins usually configure organization level integrations, while users authorize personal or user scoped connections when a tool requires user consent.

## Integration categories

| Category | Examples | Who configures it |
| --- | --- | --- |
| Identity and directory | SSO, directory sync | Admin |
| Communication | Slack, Gmail, SMS, email sending | Admin and user, depending on scope |
| Knowledge | Google Drive, Notion, MCP servers | Admin for shared connectors, user for personal authorization |
| Revenue and calls | Salesforce, Gong, Affinity | Admin |
| Custom tools | MCP servers, API backed tools | Admin or user, depending on connector scope |

## Admin setup flow

<Steps>
  <Step title="Open Admin Integrations">
    Go to Admin, then Integrations.
  </Step>
  <Step title="Connect the system">
    Follow the connector specific setup. Some connectors use OAuth, while others require credentials such as API keys, access keys, or connected app fields.
  </Step>
  <Step title="Confirm user authorization">
    For connectors that require user level OAuth, ask users to connect from Settings, then Integrations.
  </Step>
  <Step title="Grant agent access">
    Add the resulting resources or tool connections to the right project agents.
  </Step>
  <Step title="Test in chat">
    Ask the agent to list available context or perform a safe lookup before enabling automations.
  </Step>
</Steps>

## Product rule

An integration being connected does not mean every agent can use it. The agent also needs role capability, configured credentials, and resource access.

## Common launch set

For a first customer or team deployment, start with:

- SSO and directory sync, if required.
- Slack, if agents will answer or notify in Slack.
- Google Workspace, if agents need Gmail, Calendar, or Drive.
- One workflow specific system such as Salesforce, Gong, Affinity, or Notion.
- MCP only where the target system is best exposed through MCP tools.

Do not connect every possible system on day one. Connect the systems required for the first workflow, then expand.
