---
title: "User quickstart"
description: "Start using Thorin as an individual user with your personal agent, skills, and connected tools."
---

Use this path when your organization already has Thorin and you want to start working with your personal agent.

![Blank Thorin Home chat with a new chat tab and message composer.](/images/app/home-chat.png)

<Steps>
  <Step title="Sign in">
    Go to [app.thorin.com](https://app.thorin.com) and sign in with your organization account.
  </Step>
  <Step title="Open Home">
    Home opens your personal chat surface. Start with a direct request such as `Summarize my recent deal related email` or `What follow up do I owe this week?`
  </Step>
  <Step title="Check settings">
    Open Settings and confirm your profile, timezone, inbox watch, notifications, and privacy filters.
  </Step>
  <Step title="Connect user scoped tools">
    In Settings, then Integrations, connect any admin approved MCP connectors that need your personal authorization.
  </Step>
  <Step title="Install useful skills">
    Open Skills and browse the catalog. Install skills you expect to use often, or fork a skill when you want your own version.
  </Step>
  <Step title="Use skills in chat">
    Type `/` in chat to invoke a skill. Starter examples can insert a skill plus a prompt you can edit.
  </Step>
  <Step title="Review assigned work">
    Open Assigned Tasks to see tasks created by agents for you. Use filters to find open, completed, or cancelled work.
  </Step>
</Steps>

## Good first prompts

- `What do I need to follow up on today?`
- `Summarize the latest context on this account before my call.`
- `Draft a reply to this email thread and keep it concise.`
- `Find recent Slack discussion about this project and turn it into action items.`
- `/skill-name Use this on the latest Gong call for Acme.`

## Personal agent boundaries

Your personal agent acts with your allowed context. It is useful for private drafting, personal follow up, and work that depends on your access. For shared workflows, use or create a project agent so the team can inspect resources, automations, sessions, and work items together.
