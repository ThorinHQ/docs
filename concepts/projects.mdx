---
title: "Projects"
description: "Group agents, resources, and skills under a shared business process."
---

A **project** (sometimes called a *process* in older parts of the UI) is a workspace for a single business workflow — for example, "Sales Ops," "Recruiting Pipeline," or "Customer Onboarding." Everything that belongs to that workflow lives inside the project: agents, shared resources, member roles, and metrics.

Use a project when:

- More than one person needs to interact with the same agent.
- You want shared resources (a Drive folder, an inbox, a wiki) tied to a specific function.
- You want metrics and SLA tracking scoped to a team rather than to your whole org.

## Create a project

From **All Projects → New Project**:

1. Name the project (for example, `Sales Ops`).
2. Add a short description — visible to anyone browsing projects.
3. Click **Create**.

You can also create a project as part of the **Create Agent** flow if the agent doesn't have a home yet.

## Project tabs

Open a project to find:

| Tab | Contents |
| --- | --- |
| **Overview** | Description, recent activity, and quick links. |
| **Agents** | The agents owned by this project. |
| **Members** | People with a role on the project, and what role they have. |
| **Work Items** | Filtered view of [work items](/concepts/work-items) produced inside the project. |
| **Metrics** | Charts for throughput, SLA compliance, top performers, and trend lines. |

## Add members

On the **Members** tab, click **Add member** and pick a user and a role:

- **Member** — can chat with agents and view work items.
- **Manager** — everything a member can do, plus create and edit agents, skills, and responsibilities for the project.
- **Admin** — full control, including deleting the project.

Roles are scoped to the project — someone can be an admin on one project and a member on another.

## Resources

A project can own files, folders, inboxes, and email sets. Manage them in [Library](/concepts/library) or directly from the agent detail panel. Resources owned by the project are available to every agent in the project.

## Metrics

The **Metrics** tab tracks how the project is performing:

- **SLA compliance trend** — % of work items completed within their SLA over time.
- **Performer leaderboard** — agents and people who closed the most work items.
- **Category carousel** — breakdown of work items by category.

Metrics are computed from the work items the project's agents produce, so the more work items they generate, the richer the dashboard becomes.
