Work items are tasks an agent assigns to a human when it can't — or shouldn't — finish the work itself. They're how an agent escalates: a question only you can answer, an approval only you can give, a step that needs human judgment.

You'll find them under **Assigned Tasks** in the sidebar.

## What's on a work item

Every work item carries:

- **Title** — a short description of what's needed.
- **Status** — where the task is in its lifecycle.
- **Priority** — how urgent it is.
- **Assignee** — the user expected to pick it up.
- **Due date** — when it should be done.
- **Created** and **Completed** timestamps.
- **Originating agent** — the agent that handed the task off.

## Statuses

The table filter exposes the statuses available on a work item — for example open, in progress, and completed. Use the status filter to focus on what still needs your attention versus what's already done.

## Sort

Sort the table by:

- Priority
- Due date
- Created
- Completed
- Title
- Status

## Filter and search

Narrow the list with any combination of:

- **Status**
- **Priority**
- **Assignee**
- **Originating agent**

The search box matches on title.

## Open the detail panel

Click any row to open the work item's detail panel. From there you can:

- See the full context — which thread or [agent session](/concepts/agents) created the task and why.
- Update status as you make progress.
- Reassign to another user when the work belongs somewhere else.

<Tip>
  When a work item lacks context, jump to the originating session from the detail panel. The agent's reasoning usually explains exactly what it needs from you.
</Tip>
