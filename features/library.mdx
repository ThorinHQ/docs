The Library is your org-wide store of things agents read from: structured records (entities), free-form documents, folders, and projects. Anything you want an agent to use as a long-lived reference belongs here.

## Tabs

When the library page feature is enabled for your tenant, the Library page has two tabs:

- **Projects** — group related agents and resources into a workspace. Use a project to keep an initiative's agents, documents, and entities together.
- **Entities** — structured records (people, companies, deals, and other custom types) used as long-lived references across agents.

<Note>
  Tenants without the library feature flag see only the **Entities** tab. The previous standalone Entities page has folded into Library — old links to Entities still land in the right place.
</Note>

## Create from the + menu

The **+** menu on the Library page is the single entry point for creating:

- **Entities**
- **Folders**
- **Projects**
- **Agents**
- **Datasources**

Pick the kind of resource you want and fill in the form.

## Permissions

Viewing the Library is broad — most users can browse the resources they have access to. Creating and editing resources requires **operations** access on your user. If the **+** menu is missing or grayed out, ask an admin to grant operations access.

## Attach to agents

Resources in the Library are designed to be attached to agents. An agent can reference a folder of documents, an entity, a calendar, or any other library resource — and use it during reasoning, search, and tool calls. See [Agents](/concepts/agents) for how attachments work and how to add them.
