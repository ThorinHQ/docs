title: "Skills"
description: "Reusable capabilities you install on agents."
---

Skills are named, reusable capabilities you give to an agent — things like "Draft a reply", "Open a Linear issue", or "Summarize a thread". Each skill bundles an instruction with the tool access it needs, so you can hand the same capability to multiple agents without rewriting it.

You manage them from the **Skills** entry in the sidebar.

## Where skills live

The Skills page is split into three groups:

| Group | Access | What's in it |
| --- | --- | --- |
| **My skills** | `READ_WRITE` | Skills you own. You can edit, link other users, or publish them. |
| **Installed** | `READ` | Skills you've installed from the catalog. You can use them but not edit them. |
| **Catalog** | `null` | Skills that have been published org-wide but you haven't installed yet. |

The access level is per personal agent: `null` means the skill isn't installed, `READ` means it's installed read-only from the catalog, and `READ_WRITE` means you own the skill.

## Install or fork from the catalog

When you find a published skill in the **Catalog** group you have two ways to bring it onto your personal agent:

- **Install** — adds the catalog skill to your personal agent as `READ`. You get updates the owner ships, but you can't change the instruction.
- **Fork** — copies the skill into **My skills** as a new `READ_WRITE` skill you own. Edit it freely; it no longer tracks the original.

<Tip>
  Install when you trust the owner to keep the skill current. Fork when you need to tweak the instruction or the tool list for your own workflow.
</Tip>

<Steps>
  <Step title="Open Skills">
    Click **Skills** in the sidebar and switch to the **Catalog** group.
  </Step>
  <Step title="Find the skill">
    Search by skill name, description, owner, or any linked user.
  </Step>
  <Step title="Install it">
    Click **Install** on the row. The skill moves into the **Installed** group and becomes available to your personal agent immediately.
  </Step>
  <Step title="Or fork it">
    Click **Fork** instead to drop an editable copy into **My skills**.
  </Step>
</Steps>

## Owners and linked users

Every skill has a single **owner** — the user with `READ_WRITE` on it. Owners can also link additional users to a skill, which surfaces it for them in search and lets them collaborate without transferring ownership.

## Publish to the catalog

When a skill is ready for wider use, the owner can **publish** it. Publishing moves it into the org-wide catalog where any user can install or fork it onto their own personal agent.

<Note>
  Unpublishing a skill removes it from the catalog but doesn't uninstall it from anyone who already installed it.
</Note>

## Search

The search box on the Skills page looks across:

- Skill name
- Description
- Owner
- Linked users

This works inside any of the three groups, so you can scope a search to **My skills**, **Installed**, or **Catalog** depending on what you're looking for.
