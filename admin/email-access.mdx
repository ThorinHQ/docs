Email access is the policy layer that decides which mailboxes Thorin processes and which individual messages it skips. It lives at **Admin â†’ Trust Center â†’ Email Access** and applies to every agent in the tenant.

There are three things to configure: an **inbox access mode**, an optional **allowlist**, and a list of **deny rules**.

## Inbox access mode

Each tenant picks one mode. The mode controls the default for the whole org.

| Mode | Behavior |
| --- | --- |
| `ALLOW_ALL_CONNECTED` | Default. Thorin reads every Gmail mailbox a user has connected. |
| `EXPLICIT_ALLOWLIST` | Strict. Thorin only reads mailboxes you've added to the allowlist. Everyone else is silent. |

Switching from `ALLOW_ALL_CONNECTED` to `EXPLICIT_ALLOWLIST` requires explicit confirmation. As soon as you confirm, every user not on the allowlist stops being processed â€” agents won't see new mail until you add their inbox back.

## Allowlist

The allowlist is a list of inbox addresses Thorin is allowed to process. It's only meaningful in `EXPLICIT_ALLOWLIST` mode â€” in `ALLOW_ALL_CONNECTED` mode it's ignored.

Add an address from the **Allowlist** section in the Email Access tab. Remove an entry to immediately stop processing that inbox.

## Deny rules

Deny rules skip specific messages even when the inbox itself is allowed. Each rule has a **kind** that controls what it matches.

| Kind | UI label | Matches |
| --- | --- | --- |
| `INBOX` | Inbox | Skip the named address's own mailbox. Other people's email to or from this address still processes. |
| `EMAIL_PARTICIPANT` | Email participant | Skip any message this address is on, as sender or recipient. |
| `THREAD_PARTICIPANT` | Thread participant | Skip the entire thread once this address joins it. |
| `DOMAIN_INBOX` | Inbox domain | Like `INBOX`, but matches every address on the domain. |
| `DOMAIN_EMAIL_PARTICIPANT` | Email participant domain | Like `EMAIL_PARTICIPANT`, but matches the whole domain. |
| `DOMAIN_THREAD_PARTICIPANT` | Thread participant domain | Like `THREAD_PARTICIPANT`, but matches the whole domain. |
| `SUBJECT` | Subject line | Skip messages whose subject contains the configured text. |
| `PRIVILEGED_CONTENT` | Privileged content | Singleton rule controlled by the **Privileged content** toggle. |

### Privileged content

`PRIVILEGED_CONTENT` is special â€” it's a singleton rule, not something you add by hand. Toggle **Privileged content** on at the top of the Email Access tab and Thorin's classifier will skip any message it labels as privileged (legal, HR, medical, and similar). Messages it skips never enter the processing pipeline.

### Add a deny rule

<Steps>
  <Step title="Open Admin â†’ Trust Center â†’ Email Access">
    Scroll to **Deny rules**.
  </Step>
  <Step title="Click Add deny rule">
    A dialog asks for a kind and a value.
  </Step>
  <Step title="Pick a kind and value">
    For example, choose **Thread participant** and enter `counsel@example.com` to skip any thread that lawyer joins. Or pick **Subject line** and enter `[confidential]` to skip messages tagged that way.
  </Step>
  <Step title="Save">
    The rule takes effect immediately for new mail. Messages already processed aren't retroactively removed.
  </Step>
