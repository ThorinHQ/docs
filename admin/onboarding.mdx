---
title: "Workspace onboarding"
description: "Bring a team or customer workspace onto Thorin without connecting too much data."
---

Onboarding a Thorin workspace should start from one workflow, not from every possible integration.

## Recommended sequence

<Steps>
  <Step title="Define the first workflow">
    Pick a workflow with a clear owner, bounded data, and obvious success criteria.
  </Step>
  <Step title="Set up access">
    Configure sign in, SSO, and directory sync if required.
  </Step>
  <Step title="Invite admins and pilot users">
    Start with the people who will configure integrations and review agent output.
  </Step>
  <Step title="Connect minimum integrations">
    Connect only the systems needed for the first workflow.
  </Step>
  <Step title="Create a project">
    Add members, resources, and a project agent.
  </Step>
  <Step title="Test in chat">
    Run realistic examples before enabling any live triggers.
  </Step>
  <Step title="Enable narrow automation">
    Start with a low risk schedule or narrow feed trigger.
  </Step>
  <Step title="Review and expand">
    Use sessions and work items to decide whether to add more data, users, or automations.
  </Step>
</Steps>

## Launch checklist

- Admin users can sign in.
- Pilot users can sign in.
- User timezone and notification preferences are configured.
- Privacy filters are set where needed.
- Required integrations are connected.
- Project resources are scoped.
- The first agent has clear instructions.
- Automations have been tested in chat.
- Reviewers know where to inspect sessions and work items.

## Customer facing guidance

When explaining Thorin to a new team, use product language:

- "Your personal agent helps with your own work."
- "Project agents own shared workflows."
- "Resources are the context agents can use."
- "Skills are reusable methods."
- "Automations run agents on schedules or events."
- "Work items make follow up inspectable."

Avoid exposing internal implementation details or private operational runbooks in public facing onboarding materials.
