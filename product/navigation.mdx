The Thorin dashboard is organized around personal work, shared knowledge, agent operations, and admin setup.

![Thorin app navigation with personal, organization, and admin areas.](/images/app/app-navigation.png)

## Personal

| Area | Use it for |
| --- | --- |
| Home | Chat with your personal agent or another selected agent. Start new chat tabs, use `/` skills, mention resources with `@`, and review chat history. |
| Skills | Browse, install, create, fork, edit, publish, and uninstall reusable skills. |
| Automations | Review automations that agents can run from chat, schedules, or data events. |
| Switchboard | Set up and review email routing decisions when your workspace uses Switchboard. |
| Settings | Update profile, timezone, inbox watch, notifications, privacy filters, and user scoped integrations. |

## Organization

| Area | Use it for |
| --- | --- |
| Library | Browse projects and entities. Create projects, inspect project trees, add files, add data sources, and manage structured entities. |
| Assigned Tasks | Review work items assigned to users or agents. Filter by status, priority, owner, agent, and project. |
| Agent Topology | Inspect agents, resources, data sources, sessions, permissions, and project structure. |

## Admin

| Area | Use it for |
| --- | --- |
| Integrations | Configure SSO, directory sync, Slack, Affinity, Gong, Salesforce, Google Drive, and MCP connectors. |
| Trust Center | Review security and compliance information when available in your workspace. |
| Agent Topology | Admins can use topology views to inspect organization wide agent structure and access. |

## Search and context controls

Use chat when you know the result you want. Use Library or Agent Topology when you need to inspect the underlying setup. These views show which project owns an agent, which resources it can access, which automations exist, and which data source should trigger work.

<Tip>
  When debugging agent behavior, start from [Sessions and history](/agents/sessions-and-history), then use [Agent topology](/agents/topology) to inspect setup and [Capabilities and tools](/agents/capabilities-and-tools) to confirm what the agent is allowed to materialize.
</Tip>
