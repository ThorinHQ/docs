---
title: "What is Thorin?"
description: "A plain explanation of Thorin and the work it supports."
---

Thorin is a product for company agents. It connects to tools such as Slack, Gmail, SMS, Google Drive, Gong, Salesforce, Affinity, Notion, and MCP servers. It gives agents controlled access to the data and actions they need.

Use Thorin when work depends on company context and repeats often. For example, a team can use Thorin to monitor deal email, prepare account briefs, route inbound messages, track follow up, maintain shared databases, or answer questions from company knowledge.

## The product model

Thorin has a few main parts.

| Layer | What it does |
| --- | --- |
| People | Users sign in, manage personal settings, connect user scoped tools, and own personal agents. |
| Projects | Projects group agents, resources, data sources, members, and work items around a business workflow. |
| Agents | Agents chat, read context, call tools, update resources, create work items, and run automations. |
| Knowledge | Resources and data sources give agents durable context, from text files to CRM records, email feeds, and meeting transcripts. |
| Skills | Skills package repeatable instructions so users can reuse or share behavior across agents. |
| Automations | Automations let agents act on schedules or data events instead of waiting for chat. |
| Work tracking | Work items turn agent output into assigned, due, reviewable follow up. |

## Personal agents and project agents

Every user gets a personal agent. The personal agent acts for that user. It can use the data and tools the user is allowed to access. It can help with private tasks and communicate through supported channels.

Project agents belong to a project. Use them for shared workflows with shared context, owners, and repeatable automations. A deal flow agent, customer support triage agent, or account prep agent should usually be a project agent.

## What Thorin is

Thorin includes chat, but the product is broader than chat. The product also includes feeds, resources, entities, skills, triggers, sessions, work items, form factors, and permissions.

Thorin should not ingest everything by default. Good Thorin deployments start with one scoped workflow. They connect the minimum useful context. They expand access after the workflow proves reliable.

## Good first workflows

- Ask your personal agent questions across email, Slack, and connected tools.
- Create a project agent that monitors a specific inbox or data feed.
- Write a skill for a repeatable analysis or drafting workflow.
- Schedule an agent to produce a recurring digest or stale item review.
- Build a deal flow manager that classifies inbound opportunities and creates follow up work.
